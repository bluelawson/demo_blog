## 開発ルール

### コミットメッセージ

コミットするときは以下の通り、コミットメッセージにprefixをつける

- changes : → 一般的な仕様変更。機能の新規追加やロジックの変更、UIの変更、定数の変更など
- others : → ソースコードの内容の変更ではないもの。コメントの修正、Readmeの変更、import順の変更、誤字の修正など

  例：　「changes: ログアウト用APIを追加」
